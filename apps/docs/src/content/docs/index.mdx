# Welcome to The Forgebase

Authora is a powerful authentication and authorization service that provides enterprise-grade identity management solutions. It offers a comprehensive set of features for managing user authentication, authorization, and identity across your applications.

## Getting Started

<Steps>
<Step>
### Install the SDK
```bash
npm install @authora/sdk
```
</Step>
<Step>
### Initialize the Client

```typescript
import { PrivateClientSDK, PublicClientSDK } from '@authora/sdk';

// For backend services
const privateClient = new PrivateClientSDK(
  'https://api.example.com',
  'tenant-id',
  'client-id',
  'client-secret'
);

// For frontend applications
const publicClient = new PublicClientSDK(
  'https://api.example.com',
  'tenant-id',
  'client-id',
  'redirect-uri'
);
```

</Step>
<Step>
### Configure Authentication Methods
   Choose and configure your preferred authentication methods: - Password-based authentication - OAuth providers (GitHub, Google, etc.) - SAML for enterprise SSO - Email/SMS verification
</Step>
</Steps>

<Callout type="info">
  Authora provides both REST APIs and SDK clients for seamless integration with
  your applications.
</Callout>

## Core Concepts

### Tenants

Authora is built with multi-tenancy at its core. Each tenant represents an isolated environment with its own:

- Users and authentication settings
- Roles and permissions
- API keys and clients
- Custom configurations

## Integration Examples

### SDK Usage

```typescript
// User Management
await privateClient.getUsers();
await privateClient.assignRole('user-id', { role: 'admin' });
await privateClient.assignPermission('user-id', { permission: 'read' });

// OAuth Authentication
const authUrl = publicClient.getAuthorizationUrl('github');
const tokens = await publicClient.exchangeCode('auth-code');

// SAML Integration
const samlLoginUrl = publicClient.getSamlLoginUrl('provider-id');
const samlResponse = await publicClient.handleSamlCallback(samlResponse);
```

<Callout type="warning">
  Remember to securely store your client credentials and never expose private
  client keys in frontend applications.
</Callout>
